# 目录与命名规范

## 顶层信息架构（逻辑层）
- 基础：语言/浏览器/计算机基础/环境
- 面试：八股、手写题、场景题、项目面
- 业务：真实业务场景与工程实践
- 系统设计：前端架构、AI/Agent 系统设计
- 算法：数据结构与算法题

## 现有目录映射（物理层）
| 大类 | 现有目录 | 说明 |
| --- | --- | --- |
| 基础 | `前端/基础系列`、`技术/计算机基础`、`技术/配置与环境` | 前端与通用基础并行维护 |
| 面试 | `前端/面试` | 题型划分清晰，作为主入口 |
| 业务 | `前端/业务` | 业务题与工程实践沉淀 |
| 系统设计 | `前端/面试/项目面`、`前端/面试/场景题` | 逐步补齐 AI/Agent 方向 |
| 算法 | `前端/面试/算法题` | 题单与复习入口 |

## Exclude 目录用途
- 角色：草稿 + 归档区（不参与主索引与复习路线）
- 迁移规则：
  - 内容稳定后，移动到对应大类目录
  - 迁移后在 Exclude 删除原稿，避免重复
- 清理节奏：每月末抽 1 次集中整理

## 文件命名规范
- 统一命名：`领域-主题.md` 或 `主题-子主题.md`
  - 示例：`浏览器-事件循环.md`、`AI-Agent-记忆体设计.md`
- 约定：
  - MOC/入口页统一使用 `index.md`
  - 标题与文件名一致，便于检索
  - 避免无意义数字或缩写（除非业内通用）
